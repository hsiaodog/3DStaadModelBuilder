# 三視圖同步計畫(P1 / P2 / P3)

把多張 2D 圖(XY / XZ / YZ 三視圖)組合成同一個 3D 結構模型 —— 在一張圖上建節點,自動同步到其他相容的視圖。

## 共用資料模型

每一張 page 需要的設定:

| 欄位 | 在哪設 | 意義 |
|---|---|---|
| `page.plane` | 左欄 平面 dropdown / Shift+W radial picker | XY / XZ / YZ;決定 (sx, sy) → (X, Y, Z) 的對應 |
| `file.scaleRuler.ratio` | 頂部「比例尺」按鈕 | mm per px |
| `file.planeOrigin` | 頂部「座標原點」按鈕 | 2D 原點(該頁 SVG 座標),代表 3D 模型上的同一個物理點 |
| `page.z` | 左欄 第三軸位置 input | out-of-plane 軸的值;XY→Z,XZ→Y,YZ→X |

關鍵助手函式(都已實作於 `index.html`):

- `joint2DToWorld3D(file, page, joint)` —— 2D → 3D world,回 `{x, y, z, strong}`
- `world3DToJoint2D(file, page, world, opts)` —— inverse;檢查 `page.z` 與對應軸吻合
- `findCompatiblePages(world, opts)` —— 掃所有 file/page,回所有能容納此 3D 點的頁
- `inferGlobalJoint(g)` —— 根據所有綁定推算 globalJoint 的 3D 座標 + 一致性檢查
- `proposeAutoPairings(opts)` —— 配對既有節點(三視圖自動配對對話框用)

## P1:跨頁同步建點(已完成)

### 已實作

- `state.crossViewSync` (boolean) + 「跨頁同步」 toggle 按鈕(toolbar)
- `syncJointAcrossViews(joint)` —— 在「節點」工具下,如果 toggle 開啟,呼叫此函式
  1. 用當前頁算 3D world
  2. `findCompatiblePages(world)` 找相容頁(排除當前頁)
  3. 對每個相容頁:
     - 已有此 globalJoint 綁定 → 略過
     - 同位置已有節點(`< snapPx / scale`) → 重用,綁過去
     - 否則 → 反投影建新節點
  4. 全部綁到一個 globalJoint(新建或沿用 `joint.globalId`)
  5. 跑 `inferGlobalJoint` 算 3D
- HUD 訊息 `跨頁同步:新建 N 個節點(重用 M)於 K 個相容頁,全綁到 N1`

### 已知限制

- 投影位置是 raw 數值,不會吸到底圖實際交點(P2 處理)
- 桿件不同步(P3 處理)
- 相容性 tolerance 是 `max(1.0, snapPx / scale)`,無 UI 可調

## P2:bg-snap(已完成)

### 已實作

- `cacheActivePageBgSegs()` —— 把當前 active 頁的 bg 線段以世界座標(= 該頁 joint 座標)抽出存到 `page._bgSegsCache`
  - hook 在 `activatePage` 結尾,每次活躍時刷新
  - 略過 zero-length / circle / ellipse / page-bg
- `prewarmAllPagesBgCache()` —— 逐頁 activate + 抽 cache + 切回原頁
  - 「掃描所有底圖」按鈕(toolbar)觸發,withBusy spinner
  - 已快取的頁略過(避免重做)
- `snapProjectionToBgIntersection(file, page, projected, opts)` —— 用 cache 找 strict 相交點
  - bbox 預篩(線段中心 + 半長 ≤ radius)
  - O(N²) 配對,N 是預篩後的線數
  - radius 預設 `max(2.0, snapPx * 2 / scale)`
  - strict tolerance `eps = 1e-6` 線參數空間,基本就是 t/u ∈ [0, 1]
- `syncJointAcrossViews` 整合 —— 對每個 compat 頁先嘗試 snap,fallback 到 raw 反投影
  - HUD 訊息報告:新建 / 重用 / 吸到底圖交點 / 缺少快取的頁數

### 已知限制

- **未掃描的頁無法 snap** —— 第一次用要按「掃描所有底圖」一鍵預熱,或先手動切過所有頁面
- **bg 編輯後不會自動 invalidate** —— user 在 bgEdit 加線後,cache 過時。實作上每次 `activatePage` 會重建,所以只要切走再回來就會更新
- **快取大小** —— DXF 上千線會留在記憶體;單頁幾百個 segs 級別,實際成本很小
- **snap 半徑固定** —— 用 `state.snapPx`,跟一般 snap 同樣的設定;沒有獨立 P2 半徑 input

## P3:跨頁同步建桿件(已完成)

### 已實作

- `addMember(j1, j2, targetPage)` —— 多了可選 `targetPage` 參數,沒給就用 `getPage()`(向後相容)
- `syncMemberAcrossViews(j1Local, j2Local)` —— 用 globalIds 在所有其他頁找 view-binding 的對應,有就建桿件
- `addMemberInteractive(j1id, j2id)` —— 包裝 addMember:本頁建 + (若 crossViewSync) auto-promote 兩端 globalJoint + 跨頁建桿件
  - auto-promote 是借用 P1 的 `syncJointAcrossViews(joint)`,沒 globalId 的那端會先 promote
- 4 個 line-tool callsite 都改用 `addMemberInteractive`(主 wrap.click + 桿件元素 click + label-J click + outer label click)
- HUD 訊息:`跨頁同步桿件:在 N 個頁面比對,新建 K 條桿件`

### 已知限制

- **桿件刪除沒做跨頁** —— 在某頁刪桿件不會刪其他頁的對應桿件;按設計信念「降低出錯成本」,不是 silent destruction
- **不同 plane 對應失敗時無提示** —— 若兩端 globalJoint 都建好,但目標頁缺其中一端的對應,就略過該頁,沒有分項統計
- **退化情況**(g1 === g2,或 globalId 為 null):函式自然 return 空結果,不會 crash,但也沒明確 alert
- **既有桿件 dedup** —— 用 j1/j2 雙向比對,簡單可靠

## 後續可考慮(P4+)

| 方向 | 描述 |
|---|---|
| **跨頁刪除** | 選取節點 / 桿件 + Ctrl+Delete →連同所有 globalJoint 對應全刪 |
| **跨頁移動** | 在一張圖拖節點,所有對應節點同步移動(更新 globalJoint 3D + 反投影各頁) |
| **不一致檢測 panel** | `g.warnings` 集中列表 — 點擊跳到衝突的視圖,讓 user 修正 |
| **3D 預覽視窗** | 用 `state.globalJoints` + 跨視圖桿件(P3 之後)畫一個迷你 3D viewport,可旋轉檢視 |
| **匯入 STAAD 反向解** | 給現成 .std 檔,自動拆出 XY / XZ / YZ 三視圖頁面當骨架(畫底圖) |

- [ ] 比例尺(ratio)已建立
- [ ] 平面原點 已標(同一個物理點)
- [ ] `page.z` 已輸入(XY→Z,XZ→Y,YZ→X)

- [ ] P1 節點同步驗過 — 在一張圖建點,其他頁出現對應節點

- [ ] 沒有 ⚠ 警告(若有,先校正不一致的源頭)

## 設計信念

1. **2D 為 source of truth** —— 3D 是推算結果(`inferGlobalJoint`),不是另存一份。User 編輯永遠在 2D 視圖上,跨頁同步只是「同一動作多面套用」。
2. **降低出錯成本** —— 同步是 opt-in toggle,不會在 user 沒準備好時偷偷做事。
3. **失敗時 fallback,不阻擋** —— P2 找不到 bg 交點就用 raw;P3 對不到 globalJoint 就只在當前頁建。HUD 訊息要說清楚發生了什麼。
4. **校準是 page-local,不是 global** —— 多檔不同比例 / 不同原點都允許;`syncStateScaleFromActiveFile` 在頁面切換時同步 `state.scale`(顯示用)。
